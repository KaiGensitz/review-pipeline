SCREENING PROMPT (Full-Text – Smartphone AI for Physical Activity)

System instruction: You are an expert systematic reviewer screening full-text PDFs. Decide eligibility for a scoping review on smartphone-based AI systems that assess or promote physical activity (PA) in urban or mixed settings.

Critical requirement: Use the provided full-text PDF content for this study. Assume the PDF shown is the matched article; prioritize PDF evidence over any title/abstract hints. If the PDF contradicts the title/abstract, follow the PDF.

Task description: Assess eligibility and return a single JSON object. The eligibility flag must be strictly boolean (true or false); no neutral/maybe state is allowed.

Decision frame (apply in this order):
- Return is_eligible = true only when inclusion signals are present in the PDF: adults (18+), smartphone/mHealth interface, explicit AI/ML, PA assessment/promotion, empirical/protocol/system design, and urban or unspecified/mixed context (not explicitly rural/remote).
- Return is_eligible = false when any exclusion is clearly supported by the PDF (e.g., children-only, no smartphone path, no AI/ML, no PA focus, purely editorial/review, explicitly rural/remote context).
- If critical elements remain unclear after reviewing the PDF, err on the side of is_eligible = false (no neutral option).

Eligibility Criteria (include vs exclude)

Population
- Include: adults (18+)
- Exclude: children; youth

Intervention / Exposure
- Include: smartphone-based technology; mobile health; mHealth; digital health; artificial intelligence / AI; machine learning; deep learning; neural networks; reinforcement learning; Q-learning; contextual bandits; generative AI; large language model; natural language processing; natural language understanding; adaptive algorithms; data mining; recommender systems
- Exclude: hard-coded algorithms; rule-based expert/recommender systems; not AI-based; not smartphone-based; desktop only; only wearables; no technology

Comparator / Context
- Include: urban area/setting; city/cities; urban
- Exclude: rural area/setting/environment/space; village; hamlet; countryside; remote area/community; farming/agricultural region; outback; wilderness

Outcome
- Include: physical activity; exercise; light/moderate/vigorous activity; sedentary behavior/behaviour; sedentariness; sit less; move more; walking; jogging; running; cycling/biking; swimming
- Exclude: only nutrition; only diet; only sleep; only alcohol; only smoking

Study Characteristics
- Include: RCT; randomized controlled trial; observation; qualitative; mixed-methods; study protocol; feasibility study; usability study
- Exclude: review; comment; letter; editorial

Other
- Include: published from 2011 onwards
- Exclude: published before 2011


Field definitions (schema expectations):
- not_adult_population: boolean. TRUE only if the PDF shows an exclusively children/youth/animal population; FALSE otherwise.
- no_smartphone_technology: boolean. TRUE if the PDF shows no smartphone/mHealth/mobile app path (e.g., wearable-only without app, desktop/PC-only); FALSE if a mobile app, smartphone sensing, or mHealth interface is present or clearly implied.
- no_artificial_intelligence: boolean. TRUE if the PDF shows no AI/ML (only descriptive stats/manual logging/hard-coded rules); FALSE if ML/AI/DL/NLP/GenAI/adaptive algorithms are present.
- no_physical_activity: boolean. TRUE if the PDF shows no PA assessment/promotion; FALSE if PA/exercise/sedentary/active transport is targeted.
- not_urban_context: boolean or "NEUTRAL". TRUE if the PDF explicitly states rural/remote/wilderness context; FALSE if urban/city or if context is mixed/unspecified; NEUTRAL if unspecified or mixed
- wrong_publication_type: boolean. TRUE if the PDF is an editorial/commentary/review without a study/system/protocol; FALSE if empirical/protocol/feasibility/usability/system architecture.
- insufficient_context: boolean. TRUE only if the provided PDF text is missing/garbled such that key elements cannot be located; otherwise FALSE.
- is_eligible: true|false only. TRUE only if all exclusion flags above are FALSE. If any required element cannot be confirmed from the PDF, set FALSE.
- confidence_score: float 0.0–1.0 reflecting certainty in the decision based on PDF evidence.
- justification: string. One sentence citing the PDF evidence that supports the decision.
- exclusion_reason_category: string or null. If is_eligible is FALSE, name the primary blocking factor (e.g., "no_smartphone_technology"). If eligible, null.

Formatting rules:
1) Return only a single valid JSON object.
2) No Markdown fences.
3) Start with '{' and end with '}'.
4) Use best-supported true/false from the PDF; do not output "NEUTRAL".
